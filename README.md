# AI Research Toolkit

通用 AI research toolkit，用于管理非 skill 资源：可安装软件、科研图示工具链、论文图生成工具、视觉样例仓库、prompt/research writing 参考、自动科研系统参考、wrapper 和本地检查脚本。

本仓库不是具体科研项目目录，也不是 Codex skill marketplace。边界说明见 `docs/REPOSITORY_BOUNDARY.md`。

## Fresh Machine Start

```bash
git clone https://github.com/YuukiAS/AI_Research_Toolkit.git
cd AI_Research_Toolkit
bash scripts/bootstrap.sh
bin/ai-research-toolkit validate
bin/ai-research-toolkit list
bin/ai-research-toolkit sync --priority core --dry-run
bin/ai-research-toolkit doctor
```

`repo/` 中的 upstream clone 是可选本地工作副本，fresh clone 后可以不存在。需要某个资源时再用 `sync` 选择性 clone。

## Core Commands

```bash
bin/ai-research-toolkit list
bin/ai-research-toolkit validate
bin/ai-research-toolkit status
bin/ai-research-toolkit sync --id paper2any --dry-run
bin/ai-research-toolkit doctor --json
bin/ai-research-toolkit smoke
bin/ai-research-toolkit generate
```

Legacy shell entrypoints are thin wrappers:

```bash
bash scripts/check_toolkit_layout.sh
bash scripts/check_core_software.sh
bash scripts/smoke_render_diagrams.sh
```

## Layout

| Path | Purpose |
| --- | --- |
| `software/` | Installable, callable, renderable, or executable tools |
| `references/` | Reading, learning, style, prompt, writing, or system reference repos |
| `examples/` | Toolkit-owned minimal examples and smoke inputs |
| `templates/` | Toolkit-owned reusable templates |
| `scripts/` | CLI implementation, bootstrap, and wrappers |
| `inventory/` | Durable resource metadata, candidates, and retired records |
| `docs/` | Boundary docs, historical audits, and maintenance notes |
| `bin/` | Tracked wrappers and entrypoints |
| `.local/` | Ignored machine-local envs, cache, logs, smoke outputs, and state |

Each formal resource keeps this outer layout:

```text
<category>/<resource>/
  README.md
  install.md
  notes.md
  privacy.md
  examples/
  repo/
```

`repo/` is ignored and optional. Toolkit-owned files stay outside `repo/`.

## Source Of Truth

- `inventory/resources.yaml` is the only formal resource inventory.
- `inventory/candidates.yaml` records names without confirmed upstream identity.
- `inventory/retired.yaml` preserves future retired-source records.
- `RESOURCE_INDEX.md` and `LICENSE_AUDIT.md` are generated by `bin/ai-research-toolkit generate`.
- Machine-local results are written to `.local/state/resources.json` and are not committed.

## Safety Rules

- Do not use `sudo` from this toolkit.
- Do not scan raw data, NIfTI files, checkpoints, logs, submission archives, `.env*`, secrets, tokens, or credentials.
- Do not modify upstream files inside `repo/`.
- Do not reset, clean, rebase, or force-checkout upstream clones.
- Dirty upstream clones are skipped by `sync`.
